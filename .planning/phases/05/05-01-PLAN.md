**1. Database migration (`supabase/migrations/00003_watchlist.sql`):**

Create two tables:
```sql
CREATE TABLE watchlists (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT NOT NULL,
  token TEXT UNIQUE NOT NULL DEFAULT encode(gen_random_bytes(32), 'hex'),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(email)
);

CREATE TABLE watchlist_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  watchlist_id UUID NOT NULL REFERENCES watchlists(id) ON DELETE CASCADE,
  actor_id UUID NOT NULL REFERENCES actors(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(watchlist_id, actor_id)
);

CREATE INDEX idx_watchlist_items_watchlist ON watchlist_items(watchlist_id);
CREATE INDEX idx_watchlists_token ON watchlists(token);
```

Run the migration via `docker exec supabase_db_lobbylos psql -U postgres -d postgres -f -`.

**2. Data access layer (`src/lib/data/watchlist.ts`):**
- `getOrCreateWatchlist(email: string)` -- finds or creates watchlist for email, returns token
- `getWatchlistByToken(token: string)` -- returns watchlist with items and actor details
- `addToWatchlist(token: string, actorId: string)` -- adds actor to watchlist
- `removeFromWatchlist(token: string, actorId: string)` -- removes actor from watchlist
- `isWatching(email: string, actorId: string)` -- checks if email is watching actor

**3. API route (`src/app/api/watchlist/route.ts`):**
- POST: `{ email, actorId }` -- subscribes email to actor, returns `{ token, managementUrl }`
- DELETE: `{ token, actorId }` -- removes actor from watchlist

**4. WatchButton component (`src/components/watchlist/WatchButton.tsx`):**
"use client" component.
- Shows a bell/eye icon button with "Beobachten" label
- On click, opens a small inline form asking for email
- On submit, calls POST /api/watchlist with email + actorId
- Shows success message with link to management page: `/watchlist/{token}`
- Uses toast for feedback

**5. WatchlistManager component (`src/components/watchlist/WatchlistManager.tsx`):**
"use client" component for managing the watchlist.
- Receives initial watchlist data (items with actor names) as props
- Each item shows actor name, type, link to profile, and remove button
- Remove button calls DELETE /api/watchlist
- Shows empty state if no items

**6. Watchlist management page (`src/app/watchlist/[token]/page.tsx`):**
Server component that fetches watchlist by token and renders WatchlistManager.
- Shows "Meine Beobachtungsliste" heading
- Shows email (partially masked: f***@example.com)
- Renders WatchlistManager with items
- 404 if token not found

**7. Add WatchButton to profile pages:**
- Person profile: add WatchButton below PersonHeader, passing person.id
- Organization profile: add WatchButton below OrgHeader, passing org.id
- Party profile: add WatchButton below PartyHeader, passing party.id

Note: Email notifications (WATCH-02) are simulated with a console.log in the API route since we don't have a real email service in mock data mode. The UI and data layer are fully functional.
