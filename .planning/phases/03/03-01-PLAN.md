1. Create `supabase/migrations/00002_search_function.sql`:
   - CREATE OR REPLACE FUNCTION search_actors(query TEXT, max_results INTEGER DEFAULT 20)
   - Returns: id, slug, name, first_name, last_name, actor_type, canton, council, party_id, abbreviation, color, portrait_url, industry, similarity
   - Uses `similarity(a.name, query)` for ranking and `a.name % query OR a.name ILIKE '%' || query || '%'` for matching
   - Language: plpgsql
   - Apply with: `npx supabase migration up --local` (or `npx supabase db reset` if needed)

2. Create `src/lib/data/persons.ts` with server-side query functions using `createClient()`:
   - `getPersonBySlug(slug: string)`: Fetch single person from actors table by slug + actor_type='person', returns null if not found
   - `getPersonConnections(personId: string)`: Fetch all connections where source_actor_id or target_actor_id matches, with valid_until IS NULL. Also fetch the "other" actors in a second query using the collected IDs.
   - `getPersonVoteRecords(personId: string)`: Fetch vote_records joined with votes table via .select("decision, vote:votes(affair_title, vote_date, council, topic_category, description)")
   - `getPersonCommissions(personId: string)`: Fetch connections where source_actor_id = personId AND connection_type = 'membership', then fetch target actors that are commissions (filter by name containing "Kommission" or "kommission")
   - `listPersons(options: { sortBy, sortDir, party, canton, council })`: Fetch all person actors with optional filters, include connection count via a subquery or separate count query. Sort by name (default), canton, or council.

3. Create `src/lib/data/organizations.ts`:
   - `getOrgBySlug(slug: string)`: Fetch single org by slug + actor_type='organization'
   - `getOrgConnections(orgId: string)`: Same pattern as person connections
   - `listOrganizations(options: { sortBy, sortDir, industry })`: All org actors with optional industry filter

4. Create `src/lib/data/parties.ts`:
   - `getPartyBySlug(slug: string)`: Fetch single party by slug + actor_type='party'
   - `getPartyMembers(partyId: string)`: Fetch all person actors where party_id matches
   - `getPartyConnectionStats(partyId: string)`: For all party members, get all connections, group by industry. Returns Record of industry to connection count.
   - `listParties()`: All party actors, sorted by seats_nr desc

5. Create `src/lib/data/connections.ts`:
   - `getConnectionsForActor(actorId: string)`: Shared query fetching connections + related actors. Returns connections enriched with the "other" actor data.
   - Type: `ConnectionWithActor` = Connection and { otherActor: Actor }

6. Create `src/lib/data/votes.ts`:
   - `getVoteRecordsForPerson(personId: string)`: Fetch vote_records with joined vote data
   - Type: `VoteRecordWithVote` = VoteRecord and { vote: Vote }

   - Import from `@/lib/supabase/client` not server

All query functions use the server-side `createClient()` from `@/lib/supabase/server` EXCEPT search.ts which uses the browser client. Each function handles the Supabase response, returning typed data or null. Use `.single()` only for by-slug queries. Handle the bidirectional connection pattern (or filter for source/target).

Create `src/lib/utils/conflict-score.ts`:

1. Define `TOPIC_TO_INDUSTRY` mapping (Record of string to string array):
   - "Gesundheit" -> ["pharma", "medtech", "healthcare", "insurance"]
   - "Finanzen" -> ["banking", "insurance", "real_estate"]
   - "Energie" -> ["energy"]
   - "Verkehr" -> ["transport", "construction"]
   - "Landwirtschaft" -> ["agriculture", "food"]
   - "Bildung" -> ["education"]
   - "Verteidigung" -> ["defense"]
   - "Telekommunikation" -> ["telecommunications", "technology"]
   - "Umwelt" -> ["energy", "chemicals", "agriculture"]
   - "Wirtschaft" -> ["banking", "insurance", "consulting", "retail", "trade_association"]
   - "Soziales" -> ["healthcare", "ngo", "insurance"]
   - "Medien" -> ["media", "telecommunications"]

2. Define types:
   ```typescript
   type ConflictVote = {
     voteId: string;
     affairTitle: string;
     voteDate: string;
     topicCategory: string;
     decision: string;
     overlappingIndustries: string[];
   };

   type ConflictScoreResult = {
     score: number;                    // 0-100 percentage
     overlappingVotes: ConflictVote[];
     totalVotes: number;
     connectedIndustries: string[];
   };
   ```

3. Implement `calculateConflictScore(params)`:
   - Params: `personId: string, connections: Connection[], actors: Actor[], voteRecords: VoteRecordWithVote[]`
   - Step 1: From connections, find all orgs connected to this person. Get their `industry` field from the actors array.
   - Step 2: For each vote record, look up the vote's `topic_category`. Map it to industries via TOPIC_TO_INDUSTRY.
   - Step 3: Check if any mapped industries overlap with the person's connected industries.
   - Step 4: Score = (votes with overlap / total votes) * 100
   - Step 5: Return ConflictScoreResult with the score, list of overlapping votes with details, total vote count, and list of connected industries.

4. Export `TOPIC_TO_INDUSTRY` for use in the methodology explanation UI.

This is a pure function with no side effects -- easy to test and explain. The algorithm is intentionally simple and transparent per PROF-10 (methodology transparency requirement).
